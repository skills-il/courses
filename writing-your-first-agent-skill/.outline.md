# writing-your-first-agent-skill — outline

**Audience:** developers, beginner-to-intermediate
**Wave:** 1 (plan Task 37)
**Differentiation from `skills-il-skill-creator` skill (137 installs):** that skill's "Do NOT" clause is "Do NOT use for editing existing skills or creating skills for non-skills-il platforms." It is a WORKFLOW tool that scaffolds a new skill via interactive Q&A. The course is the EDUCATION layer: what a skill IS, why it works, how to decide what to write, and what makes one good vs bad. Course teaches concepts and examples; skill executes the scaffolding workflow.

**Recommended skills (catalog coupling):**
- `skills-il-skill-creator` (137 installs) — the scaffold tool
- `israeli-id-validator` (123 installs) — small focused exemplar to study (scripts/ usage)
- `hebrew-seo-geo-toolkit` (220 installs) — complex bilingual exemplar
- `israeli-phone-formatter` (99 installs) — another small utility exemplar

## Chapters (6)

### Chapter 1: What is a skill, really? (Mental model)
- A skill = instructions + optional code the LLM reads on demand
- Skill vs MCP server vs system prompt vs CLI tool — when each fits
- The Anthropic Claude Skills spec (SKILL.md + frontmatter)
- The skills-il extension: metadata.json + SKILL_HE.md
- When NOT to write a skill (decision tree: skill / MCP / pure tool / nothing)
- Most common mistake: writing a skill where an MCP would fit better
- Skill mention: `skills-il-skill-creator` (the scaffold tool)

### Chapter 2: Anatomy of a great SKILL.md
- Three mandatory frontmatter fields: name, description, license
- The description is the LLM's routing input — it's marketing AND specification
- The "Use when..." pattern and the "Do NOT use for..." pattern (the Do-NOT clause is critical)
- Body structure: clear scope, decision rules, worked examples, anti-patterns
- The "show your work" principle: every claim in the body must be either intuitive OR derivable from the body itself
- Study example: `israeli-id-validator` — small focused skill, clean scope
- Most common mistake: vague description ("a skill for Israeli things")

### Chapter 3: metadata.json + bilingual content (the skills-il extension)
- Why skills-il uses metadata.json instead of nested YAML (Claude Desktop rejection)
- Fields: display_name, display_description, audience, level, tags (he+en), supported_agents
- The SKILL_HE.md companion file
- Hebrew-natural language for description (avoid calques)
- Bilingual tags: how to map Hebrew tags to English equivalents for filtering
- Study example: `hebrew-seo-geo-toolkit` (220 installs) — high-quality bilingual
- Most common mistake: literal English-to-Hebrew translation that reads broken to Israeli readers

### Chapter 4: references/ vs scripts/ — when to use which
- references/ = static files the LLM reads on demand (markdown, JSON, sample data)
- scripts/ = executable code the LLM CAN run (validators, parsers, deterministic computations)
- Decision tree:
  - Deterministic compute that the LLM should not redo → scripts/
  - Lookup table or reference data → references/
  - Sample/template/example → references/
  - Anything requiring an API key or network call → scripts/ (with env-var contract)
- File size and structure recommendations
- Study example: `israeli-id-validator`'s check-digit script (correct scripts/ use)
- Most common mistake: pasting a long lookup table into SKILL.md body instead of references/

### Chapter 5: Publishing to the skills-il catalog (publication flow)
- Pick the right category repo (12 repos, mapping from topic to category)
- Scaffold via `skills-il-skill-creator` interactive flow
- Local validation: lint, schema check, em-dash gate, calque grep
- Push to the category repo (`git push origin master`)
- Sync pipeline (`pnpm sync-skills`) reads the repo and seeds the catalog DB
- Trust pipeline: what signals it scans (signed releases, MFA, branch protection, scorecard) and how to score "verified"
- Iterating: version bumps, changelog discipline
- Most common mistake: pushing without updating metadata.json version, breaking the changelog

### Chapter 6: Common rejection reasons + a 10-minute pre-publish checklist
- 10 reasons a skill gets bounced from review
- The 10-minute pre-publish checklist (read aloud, grep gates, link checks)
- The "would a smart user be better off with the chatbot's default answer?" test (if yes, don't ship)
- When to ship a skill vs an MCP server (decision tree by use case)
- Closing: where to find example skills to learn from (point to top-installed in each category)
