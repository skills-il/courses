# ai-agents-for-everyone — outline

**Audience:** non-technical, beginner
**Wave:** 1 (plan Task 39 variant — original, not fork)
**Differentiation from existing catalog:** No existing course/skill teaches "what is an AI agent, when to use one, how to set one up without writing code." Closest skills (hebrew-chatbot-builder, n8n-hebrew-workflows) are developer-focused or specific use-cases. This course IS irreplaceable — it's the conceptual gateway for non-technical Israeli users.

**Recommended skills (catalog coupling):**
- `n8n-hebrew-workflows` (37) — for graduating to no-code automation
- `make-com-israeli-automations` (32) — alternative no-code platform
- `israeli-personal-assistant` (56) — Israeli daily-context reference companion
- `israeli-customer-support-automator` (75) — concrete deployed agent example

## Chapters (6)

### Chapter 1: What is an AI agent (in plain language)
- Difference between chatbot (answers one Q) and agent (does a multi-step task)
- Three real Israeli examples: accountant prep, lawyer drafting, marketing research
- The "stop typing into a chat box" moment when you realize you need an agent
- Most common mistake: trying to make ChatGPT do everything in one mega-prompt

### Chapter 2: The three platforms for non-developers (Claude Projects, ChatGPT GPTs, Gemini Gems)
- Plain-language tour of each
- When each fits (Claude for long-form thinking; ChatGPT GPTs for structured templates; Gemini Gems for Google ecosystem)
- Hebrew support comparison (which one writes the best Hebrew, which understands Hebrew best)
- Israeli context: pricing in ₪, language defaults
- Step-by-step: how to set up your first one (no code, web UI only)
- Most common mistake: paying for premium before exhausting the free tier

### Chapter 3: Prompt patterns that turn a chat into an agent
- The "role + task + constraints + format" pattern
- The "step-by-step thinking" pattern (asking for breakdown)
- The "critique mode" pattern (have the AI critique its own output)
- The "persona" pattern (Israeli tax advisor, marketing manager, etc.)
- Hebrew-specific: when to switch language mid-prompt, when to ask for both languages
- Most common mistake: vague tasks ("help me with X" instead of "draft a 200-word X for audience Y in tone Z")

### Chapter 4: Real Israeli use cases worked end-to-end
- Use case 1: Freelancer accountant prepares monthly VAT report draft
- Use case 2: Solo lawyer drafts a standard service agreement
- Use case 3: Marketing manager researches Israeli competitors
- Use case 4: Small business owner sets up Hebrew customer-email templates
- Each: the actual prompts, what to do with the output, what to verify
- The "this is a draft, you are the editor" principle

### Chapter 5: When AI agents fail (and what to do)
- 5 failure modes: hallucinated facts (the freelancer-year-style invented threshold), training-cutoff stale data, missing context, no sources, biased or weird Hebrew output
- The "verify before you trust" protocol (cross-check with a primary source, ask the agent to cite, ask "are you sure")
- When to escalate to a real human (accountant, lawyer, doctor, licensed advisor)
- The "not advice" rule — AI is research support, not a regulated professional

### Chapter 6: Graduating to no-code automation
- When chat hits its limit (repeating tasks, schedule-driven, multi-app data)
- Introduction to no-code tools: n8n, Make.com, Zapier — what each does
- The tradeoff: more setup time, but runs without you sitting there
- Israeli use cases that fit each tool (n8n: webhook chains; Make.com: spreadsheet automations; Zapier: cross-app)
- Recommended skills: n8n-hebrew-workflows (37 installs), make-com-israeli-automations (32 installs)
- Most common mistake: skipping the chat-based phase and trying to build the automation first (you learn the workflow by doing it manually first)
